## Reading these

Each example walkthrough has the same structure:

1. **The operator** — who they are, what constrains them
2. **Pack selection** — which packs and why
3. **Roster** — which agents to activate
4. **Rhythms** — the schedule
5. **A week in the life** — actual commands by day, expected outputs
6. **Failure modes** — what to do when something goes sideways

Use them as templates. Steal the manifest, steal the roster, steal the rhythm set. Customize the tokens and you're 80% there.
