Local Service Business

The operator

Rosa owns a 6-person physiotherapy clinic. She handles ops, hiring, marketing, and a chunk of patient care. Pain points:

  • Patient inquiries fall through cracks (inbox chaos)
  • Same operational questions re-answered every month (no SOPs)
  • Cashflow forecast is “Rosa’s intuition” — sometimes scary
  • Wants to add a second location but can’t figure out if it’s affordable

She doesn’t write code. The clinic computer runs Windows. She’s comfortable in VS Code because her nephew set it up for her once.

founderstaff for non-tech founders

Rosa’s workflow proves the design: zero Python dependencies, no terminal mastery required after setup, every interaction is natural-language chat with named agents. Setup is a one-time pip-free, npm-free git clone + python relearn.py — anyone who can install VS Code can do this.

Pack selection

Skipping content (not a content-led business), delivery (no clients in the consultant sense), research (limited need), product (services, not product).

Rhythms

SOP authoring — the highest-leverage skill

Rosa’s clinic has a hundred small workflows that live only in her head: how to onboard a new patient, what to say when someone asks about insurance, how to handle a no-show, end-of-day cash reconciliation. Every time she’s away or a staffer is new, things slip.

Returns a structured SOP in 01-operations/sops/new-patient-intake.md:

  • Trigger (when this SOP applies)
  • Inputs needed
  • Step-by-step (numbered, with decision points)
  • Outputs / deliverables
  • Edge cases
  • “If this goes wrong” section

She prints it, laminates it, sticks it next to the front desk. Within 3 months she has 18 SOPs. The clinic runs without her present for the first time.

Cashflow visibility

She provides current month revenue, recurring costs, payment terms. Returns:

  • 12-month forecast with low/expected/high scenarios
  • Runway in each scenario (months until cash-negative)
  • Top 3 sensitivities (e.g., “if patient throughput drops 15%, runway compresses to 4 months”)

Suddenly she can answer “can we afford the second location?” with numbers.

Second-location council

3 agents convene:

  • finance-analyst — runs the model: capex, ramp time to breakeven, hiring costs
  • brand-strategist — does the second location dilute or extend the brand? Same neighborhood (cannibalize) or different (extend)?
  • chief-of-staff — does Rosa have the operational bandwidth to oversee two locations given current SOP coverage?

Round 1 (isolated): each gives independent take. Round 2 (cross-read): each updates given others. Synthesis:

Rosa logs this as ADR in 09-meta/decisions/2026-06-15-defer-second-location.md. Six months later she revisits with updated numbers.

Failure modes

SymptomFix
Drafts sound too corporate / not like RosaAdd a voice file (mimic content-creator pattern) with 3 of her actual emails as exemplars
/inbox-triage mis-categorizingRefine the triage rules in 01-operations/inbox-rules.md (she can edit; agent reads)
Cashflow model feels disconnected from realityUpdate 07-finance/inputs.md monthly with actuals; forecast quality is input-quality
SOPs going staleSchedule /sop-author audit existing SOPs for drift quarterly

What this OS produces in 1 year

  • 18 → 35 SOPs; clinic operates 4 weeks/year without Rosa present
  • Inbox: zero-fallthrough on patient inquiries; conversion from inquiry → booked appt up ~20%
  • Cashflow: monthly forecasts; no surprise tight months
  • Second location: opened in Q1 of year 2, profitable in 9 months (vs original “I have no idea” planning)
  • Time saved: ~6 hours/week of admin she used to do; spent on patients or family
